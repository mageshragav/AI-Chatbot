# **Model: MetalType**

## **Model Definition**

A small reference table representing predefined metal categories used throughout the system.
It stores one of three fixed metal types (**Gold**, **Silver**, **Platinum**) and is primarily used as a lookup table for manufacturing, product configuration, and inventory workflows.

---

## **Fields**

| Field Name   | Type          | Description                                                      |
| ------------ | ------------- | ---------------------------------------------------------------- |
| `id`         | AutoField     | Primary key                                                      |
| `metal_name` | CharField     | Stores the metal type value (1 = Gold, 2 = Silver, 3 = Platinum) |
| `is_active`  | BooleanField  | Indicates whether this metal type is enabled for use             |

---

## **Relations**

* No foreign keys inside this model.
* Used as a **foreign key target** by manufacturing, product, and inventory tables.
* Join paths typically originate from other models pointing to `MetalType` for classification.

---

## **Common Business Use Cases**

* Provide selectable metal types in product/manufacturing forms
* Validate metal type selection in workflows
* Categorize items, batches, or products based on metal
* Enable/disable metal categories without altering schema

---

## **Notes**

* Acts as a static lookup table with three predefined values
* Indexed on `metal_name` and `(is_active, metal_name)` for fast filtering
* Common join target when enriching product or manufacturing records

---

# **Model: Project**

## **Model Definition**

Represents a configurable project entity used across the system for organizing designs, layouts, and associated metal categories.
Supports activation control through `StatusMixin` and automatic timestamp tracking through `TimestampMixin`.

---

## **Fields**

| Field Name     | Type         | Description                                            |
| -------------- | ------------ | ------------------------------------------------------ |
| `id`           | AutoField    | Primary key                                            |
| `project_name` | CharField    | Unique name used to identify the project               |
| `description`  | TextField    | Optional descriptive text describing the project       |
| `metal_type`   | ManyToMany   | Associates the project with multiple metal types       |
| `layout`       | BooleanField | Indicates whether the project has an associated layout |
| `is_active`    | BooleanField | Indicates whether the project is enabled for use       |

---

## **Relations**

* **Many-to-Many with `MetalType`** stored in the join table:
  **`projects_project_metal_type`**

  | Column Name    | Description                       |
  | -------------- | --------------------------------- |
  | `project_id`   | FK → `projects_project.id`        |
  | `metaltype_id` | FK → `manufacturing_metaltype.id` |

* Inherits additional fields from mixins:

  * `created_at`, `updated_at` (from `TimestampMixin`)
  * `is_active` (from `StatusMixin`)

---

## **Common Business Use Cases**

* Grouping product designs or manufacturing configurations
* Associating a project with one or more metal categories
* Filtering active or inactive project definitions
* Retrieving projects based on metal type criteria
* Managing layout-specific project workflows

---

## **Notes**

* Many-to-many assignments are stored in `projects_project_metal_type`
* `project_name` is unique and indexed for fast lookups
* Prefetching (`prefetch_related`) recommended when loading metal types
* Frequently used in design, workflow, and configuration modules

  | `size_id`   | FK → `manufacturing_size.id`  |
